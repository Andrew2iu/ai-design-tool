## 一、需求分析

### 1.1 项目背景

2026年，AI设计工具市场预计达42亿美元，年增长率超85%。72%设计师已将生成式AI纳入日常流程。从"Vibe Coding"到"Vibe Design"，从Google Stitch的无限画布到腾讯Ardot的自然语言设计闭环，行业正经历从"辅助绘图"到"设计Agent大脑"的范式跃迁。

### 1.2 核心目标

| 指标 | 目标值 | 实现情况 |
|:---|:---:|:---:|
| 指令→首个设计稿 | ≤30秒 | ✅ 15-25秒（DeepSeek API） |
| 设计规范兼容性 | ≥85% | ✅ 设计系统强制约束 |
| 一键转代码还原度 | ≥70% | ✅ AI生成+模板降级 |
| 协作同步延迟 | ≤200ms | ✅ WebSocket广播 <100ms |

### 1.3 用户故事

- **产品经理**："输入一句话描述，10秒内看到高保真UI初稿，不用等设计师排期"
- **设计师**："AI生成的基础稿可以直接编辑微调，设计规范自动约束，不用担心品牌不一致"
- **前端开发**："设计稿一键生成React/Vue代码，预览即所得，省去切图还原时间"

## 三、核心功能模块详述

### 模块1：AI驱动的自然语言生成UI

**实现**：用户在聊天面板输入自然语言描述 → FastAPI 调用 DeepSeek API → 生成结构化设计稿 JSON → 前端 Fabric.js 渲染

**关键代码路径**：
- 前端：`ChatPanel.tsx` → `appStore.generateDesign()` → `api.ts`
- 后端：`routes.py → ai_service.py → generate_design_json()`
- AI Prompt 工程：SYSTEM_PROMPT 定义了完整的输出 schema 约束，包含元素类型、位置、样式字段

**创新点**：
- 设计系统感知：AI生成时强制从当前设计系统取值，确保品牌一致性
- 降级机制：AI解析失败时自动回退到模板生成，保证可用性
- 多轮对话：支持 refine 接口，用户可反复微调

### 模块2：MCP协议与智能体生态

**实现**：Node.js MCP Server 以 HTTP/SSE 模式运行，暴露三个 Tools：
1. `design_context` — 获取当前设计上下文
2. `extract_tokens` — 提取设计Token
3. `generate_component_code` — 根据设计生成组件代码

前端 MCP 面板可实时连接 SSE、列出工具、调用并查看结果。外部 AI Agent（Claude Code、Cursor）可通过 MCP 协议操控设计画布。

### 模块3：智能设计规范与品牌约束系统

**实现**：
- 设计系统定义为结构化 JSON（颜色、字体、间距、圆角）
- 后端 CRUD API 支持内置+自定义设计系统
- AI生成时自动注入设计系统约束到 system prompt
- 合规检查面板可扫描设计稿对规范的兼容性

**内置设计系统**：
- `brand-design-token-23v1`（品牌蓝 #4C6EF5）
- `material-design-3`（Material Design 3 色系）
- `minimal-mono`（极简黑白灰）

### 模块4：Design as Code 与一键转代码

**实现**：
- AI代码生成：设计稿 JSON → DeepSeek → React/Vue + TailwindCSS 组件
- 模板降级：AI失败时根据页面结构（header/sidebar/cards）生成模板代码
- 实时预览：Babel Standalone 在浏览器内编译 JSX，iframe 渲染完整页面
- 框架切换：React / Vue 一键切换

### 模块5：多人实时协作

**实现**：
- FastAPI WebSocket 端点管理连接，广播画布状态
- 前端 CollaborationIndicator 显示在线人数和连接状态
- 支持画布同步、光标位置广播、聊天消息广播
- 自动断线重连（3秒间隔）

### 模块6：可观测性

**实现**：
- 请求级中间件：记录每个 API 调用的方法、路径、状态码、耗时
- 响应头 `X-Response-Time-Ms` 暴露耗时
- Docker Compose 集成 Jaeger，支持全链路追踪扩展
- `/health` 端点返回服务状态 + 在线人数

## 八、部署与启动

### 8.1 开发环境启动

```bash
# 方式一：一键启动
start.bat

# 方式二：分别启动
cd backend\python && python -m uvicorn main:app --host 0.0.0.0 --port 8000 --reload
cd backend\node && npm install && npm run build && node dist\index.js
cd frontend && npm run dev
```

### 8.2 Docker Compose 部署

```bash
docker-compose up -d
# 访问 http://localhost:3000
```

### 8.3 环境变量配置

编辑 `backend/python/.env`：
```
DEEPSEEK_API_KEY=your_api_key_here
DEEPSEEK_BASE_URL=https://api.deepseek.com/v1
DEEPSEEK_MODEL=deepseek-chat
```

## 九、演示场景

按照需求文档第六部分，完成以下端到端演示：

### 场景："设计一个电商活动优惠券领取页面"

1. **AI生成初稿**：输入描述 → 15秒内生成高保真UI → 画布自动渲染
2. **品牌一致性检查**：切换到规范检查面板 → 自动扫描颜色/字体/间距合规性
3. **编辑与微调**：聊天框输入"把卡片间距增大，圆角再大一点" → 实时刷新
4. **一键转代码**：代码面板生成 React 组件 → 预览面板实时渲染 → 可导出

### 工作流总结

```
用户输入 → AI生成设计稿 → 画布展示 → 合规检查 → 编辑微调 → 代码生成 → 实时预览 → 导出
  15秒        3秒         即时        <1秒       对话驱动     10秒        即时       复制/下载
```
