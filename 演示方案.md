# 🤖 AI Design Tool — 系统演示方案

---

## 一、演示前准备（5 分钟）

### 1.1 启动系统
双击项目根目录的 `start.bat`，等待三个窗口都出现：
```
[后端]  http://127.0.0.1:8000       ← FastAPI
[MCP]   http://127.0.0.1:3001       ← MCP 服务
[前端]  http://localhost:3000        ← React 前端
```

### 1.2 打开浏览器
访问 **http://localhost:3000**，确认页面正常加载。

### 1.3 备选方案（如果 AI 生成失败）
准备一张截图或提前生成好的设计稿，作为兜底展示。

---

## 二、演示流程（共 10-12 分钟）

---

### 🎬 第 1 步：开场介绍（1 分钟）

**演示操作**：展示 `README.md` 或项目结构图

> 我做的项目叫 **AI Design Tool**，是一个面向 AI 原生时代的产品设计工具。
>
> **核心功能**：用户只需用自然语言描述需求，AI 就能自动生成可编辑的可视化设计稿。
>
> **技术栈**：前端 React + TypeScript + Fabric.js 可视化画布，后端 FastAPI + DeepSeek 大模型，支持设计系统管理、组件库、代码生成预览和实时协作。

---

### 🎨 第 2 步：AI 对话生成设计稿（3 分钟）⭐ 核心演示

**演示操作**：
1. 点击左侧 **「AI 助手」** tab
2. 在输入框输入：

```
设计一个电商活动优惠券领取页面，红色调，圆角风格
```

3. 点击发送，等待 AI 生成（约 5-10 秒）
4. 生成完成后，画布自动展示设计稿

**讲解要点**：
> 我输入了"设计一个电商活动优惠券领取页面，红色调，圆角风格"——注意这里用了自然语言描述，没有写任何代码。
>
> AI 自动识别了「红色调」→ 映射到色值 #E34D59，识别了「圆角风格」→ 自动设置圆角，识别了「优惠券」→ 生成对应的卡片组件。
>
> 这是系统的核心能力：**Prompt-to-Design，一句话生成设计稿**。

**预期效果截图位置**：截取 AI 对话 + 画布的完整界面

---

### ✏️ 第 3 步：画布编辑（2 分钟）

**演示操作**：
1. 在画布上点击某个元素（如标题文字）
2. 右侧属性面板会显示该元素的属性
3. 修改文字内容、颜色、大小
4. 拖拽移动元素位置
5. 缩放调整元素大小

**讲解要点**：
> AI 生成的设计稿不是静态图片，而是**完全可编辑的矢量元素**。
>
> 每个矩形、文字、按钮都是独立的 Fabric.js 对象，可以拖拽、缩放、改颜色、改文字，就像用 Figma 一样。

---

### 💻 第 4 步：代码生成与预览（2 分钟）⭐ 第二大亮点

**演示操作**：
1. 点击顶部 **「代码」** tab
2. 选择输出框架：**React + TailwindCSS**
3. 点击 **「生成代码」**
4. 切换到 **「预览」** 子 tab
5. 展示实时渲染的 React 组件预览

**讲解要点**：
> 设计稿不是终点——系统可以把设计直接**转换成可运行的代码**。
>
> 我选了 React + TailwindCSS，生成的就是标准的 JSX 代码，包含完整的 className、样式。
>
> 预览模式下，代码通过 Babel 在浏览器端实时编译并渲染，可以看到真实效果。
>
> 这打通了「设计 → 开发」的最后一公里。

---

### 🧩 第 5 步：设计系统 & 组件库（1 分钟）

**演示操作**：
1. 点击 **「设计系统」** tab，展示预设的颜色、字体、间距 token
2. 点击 **「组件库」** tab，展示可拖拽的预设组件

**讲解要点**：
> 系统内置了设计系统管理器和组件库。设计系统定义了颜色、字体、间距等设计 token，保证 AI 生成的设计稿风格统一。
>
> 组件库支持拖拽到画布，可以快速搭建原型。

---

### 🤝 第 6 步：协作功能（1 分钟，可选）

**演示操作**：
1. 打开浏览器无痕窗口，访问同一地址
2. 在第一个窗口编辑画布
3. 展示第二个窗口同步更新

**讲解要点**：
> 通过 WebSocket 实现了实时多人协作。多个用户可以在同一画布上协同编辑，改动实时同步。

---

### 📊 第 7 步：总结（1 分钟）

> 总结一下，这个项目实现了四个核心能力：
> 1. **Prompt-to-Design** — 自然语言驱动 AI 生成设计
> 2. **可视化编辑** — Fabric.js 画布，像 Figma 一样编辑
> 3. **代码生成** — 设计稿一键转 React/Vue 代码并实时预览
> 4. **设计系统** — 内置 token 管理和组件库
>
> 技术栈覆盖 React、TypeScript、FastAPI、WebSocket、大模型 API 调用，是一个完整的全栈项目。

## 三、录屏方案

### 推荐工具
| 工具 | 平台 | 优点 |
|------|------|------|
| **OBS Studio** | Windows/Mac/Linux | 免费，专业级，可加画中画摄像头 |
| **Windows 自带录屏** | Win10/11 | 按 Win+G 呼出，零配置 |
| **ScreenToGif** | Windows | 轻量，出 GIF 方便放 README |

### 录制建议
- 分辨率：1920×1080
- 放大浏览器缩放至 125%-150%，让评委看清文字
- 录完整流程后剪掉等待 AI 生成的空白时间
- 加简单的字幕或箭头标注关键操作
- 最后导出 MP4，控制在 5-8 分钟

## 四、常见演示问题 FAQ

### Q1：AI 生成很慢或失败？
**应对**：切换到「设计系统」tab 展示预设模板；或展示上一次成功生成的截图。

### Q2：代码预览白屏？
**应对**：刷新页面重试；或直接展示生成的代码文本（代码面板本身也是亮点）。

### Q3：评委问「你这个跟 Figma AI 有什么区别」？
**回答**：Figma AI 是封闭的商业产品。我们的项目证明了「Prompt-to-Design → 可编辑画布 → 代码生成」这条链路是可行的，而且完全开源，可以作为团队自建设计工具的基础。
